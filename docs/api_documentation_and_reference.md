# 🔌 JanAI — API Documentation & Technical Reference Guide

This document provides a complete **API Reference & Technical Concepts Guide** for **JanAI**. 

It explains **what** each technical concept means (TTS, STT, LLM, RAG, WebRTC, DSP, TwiML), **why** every external API was chosen, **what exact data** it provides, and **how** it connects to the system architecture.

---

## 1. 📘 Core Technical Concepts & AI Terminology

```
       ┌─────────────────────────────────────────────────────────────┐
       │                   VOICE ASSISTANT PIPELINE                  │
       └─────────────────────────────────────────────────────────────┘
                                      │
  [Human Voice Audio]                 ▼
           │             1. STT / ASR (Speech-to-Text)
           │             Converts spoken audio waveform to text transcript
           ▼
     [Text String] ──────2. Intent Router & RAG (Retrieval-Augmented Generation)
           │             Retrieves facts from DynamoDB Titan Vector DB
           ▼
     [Context + Query] ──3. LLM (Large Language Model - Bedrock Nova-Lite)
           │             Generates natural response tokens
           ▼
     [Response Text] ────4. TTS (Text-to-Speech - Cartesia / Sarvam)
           │             Synthesizes text into high-fidelity voice audio
           ▼
 [Audio Output Stream]
```

### A. 🎙️ STT / ASR (Speech-to-Text / Automatic Speech Recognition)
* **What it is:** Software that analyzes incoming digital sound waveforms from a microphone or phone line and transcribes them into written text strings.
* **Key Metrics:**
  * **Word Error Rate (WER):** Percentage of incorrect words transcribed.
  * **Acoustic Model:** Neural network trained on specific accents, background noise, and dialects.
* **In JanAI:** Driven by **Sarvam Saarika v1** (tuned for 10+ Indian languages) and **Twilio Speech Recognition**.

---

### B. 🗣️ TTS (Text-to-Speech / Speech Synthesis)
* **What it is:** Generates human-sounding voice audio from written text generated by the AI.
* **Key Metrics:**
  * **TTFA (Time to First Audio):** Delay between sending text and receiving the first playable audio chunk.
  * **Sampling Rate:** 16kHz / 24kHz audio fidelity. Higher rates sound clearer but increase bandwidth.
  * **Audio Codecs:** Opus (compressed web streaming) vs PCM/WAV (raw uncompressed).
* **In JanAI:** Driven by **Cartesia Sonic-3** (ultra-fast 40ms TTFA) and **Sarvam Bulbul v2** (regional female/male personas).

---

### C. 🧠 LLM (Large Language Model)
* **What it is:** Transformer-based neural network trained on vast text data. It acts as the central "brain" for reasoning, conversation, and language translation.
* **Key Metrics:**
  * **TTFT (Time to First Token):** Delay before the model starts streaming output text.
  * **Context Window:** Amount of text memory the model can analyze per request.
* **In JanAI:** Driven by **Amazon Bedrock Nova-Lite (`amazon.nova-lite-v1:0`)** with persona prompts (`Arya`, `Hitesh`, `Vidya`).

---

### D. 📚 RAG (Retrieval-Augmented Generation)
* **What it is:** Grounding mechanism that prevents the LLM from hallucination. It searches a vector database of official government scheme documents and feeds exact facts into the LLM context.
* **Vector Embeddings:** Converting text sentences into 1024-dimensional floating-point array coordinates (`amazon.titan-embed-text-v2:0`) to measure semantic similarity using cosine distance.

---

### E. 🌐 WebRTC & DSP (Web Real-Time Communication & Digital Signal Processing)
* **WebRTC:** Open-source protocol enabling direct peer-to-peer real-time audio/video streaming in browsers without plugins.
* **DSP (Digital Signal Processing):** OS/Browser hardware filters enforcing `echoCancellation` (prevents AI audio looping back), `noiseSuppression` (filters background fan/traffic noise), and `autoGainControl` (equalizes volume).

---

### F. 📞 TwiML (Twilio Markup Language)
* **What it is:** XML-based instruction standard (`<Gather>`, `<Say>`, `<Play>`, `<Redirect>`, `<Pause>`) returned by AWS Lambda to instruct Twilio how to handle a live phone call.

---

## 2. 🔌 External APIs & Services Directory

### 1. 📞 Twilio Telephony & WebRTC API
* **Provider:** Twilio Inc.
* **SDK / Endpoint:** `twilio.rest.Client`, `https://api.twilio.com/2010-04-01/Accounts`
* **Why We Use It:** Handles incoming/outbound PSTN phone calls (`+1 831 253 0646`) and issues JWT Access Tokens for WebRTC browser calls.
* **What It Provides:**
  * Outbound Call Trigger (`POST /call/initiate` $\rightarrow$ `twilio_client.calls.create()`).
  * Live Call Controls via TwiML XML responses.
  * WebRTC Token Generation (`GET /voice/token` $\rightarrow$ `AccessToken`, `VoiceGrant`).

---

### 2. 🧠 Amazon Bedrock Runtime API (Nova-Lite LLM)
* **Provider:** Amazon Web Services (AWS)
* **Model ID:** `amazon.nova-lite-v1:0`
* **Why We Use It:** Primary conversational LLM. Hosted on AWS us-east-1 with ultra-fast latency (~250ms TTFT) and native Indian compliance support.
* **What It Provides:** Multi-turn conversational streaming (`bedrock.invoke_model_with_response_stream`).

---

### 3. 📐 Amazon Bedrock Titan Embeddings API
* **Provider:** Amazon Web Services (AWS)
* **Model ID:** `amazon.titan-embed-text-v2:0`
* **Why We Use It:** Vector embedding generator for government scheme documents.
* **What It Provides:** Converts text chunks into 1024-dimensional numerical vectors stored in DynamoDB `janai_vectors`.

---

### 4. ⚡ Cartesia Sonic-3 Speech Synthesis API
* **Provider:** Cartesia AI
* **Endpoint:** `https://api.cartesia.ai/tts/bytes`
* **Why We Use It:** Primary low-latency TTS provider. Delivers **40ms TTFA**, cutting speech latency by 85%.
* **What It Provides:** Natural Indian English and Hindi voice audio synthesis returning binary audio bytes.

---

### 5. 🇮🇳 Sarvam AI Speech APIs (Saarika & Bulbul)
* **Provider:** Sarvam AI (India)
* **Endpoints:**
  * STT: `https://api.sarvam.ai/speech-to-text` (`sarvam-saarika-v1`)
  * TTS: `https://api.sarvam.ai/text-to-speech` (`bulbul:v2`)
* **Why We Use It:** Specialized regional Indian language speech processing tuned for Hindi, Marathi, Tamil, Telugu, Kannada, and Bengali dialects.
* **What It Provides:** Regional audio transcription and high-quality localized voice synthesis.

---

### 6. 🌾 data.gov.in Agmarknet Mandi API
* **Provider:** National Informatics Centre (NIC) / Govt of India
* **Endpoint:** `https://api.data.gov.in/resource/9ef84268-d588-465a-a308-a864a43d0070`
* **Why We Use It:** Fetches real-time daily agricultural market (Mandi) prices across Indian states.
* **What It Provides:** Live JSON records with `commodity`, `market`, `state`, `modal_price` (in ₹/quintal), and `arrival_date`.

---

### 7. 🔍 DuckDuckGo Web Search API (Fallback Search)
* **Provider:** DuckDuckGo
* **Endpoint:** `https://html.duckduckgo.com/html/`
* **Why We Use It:** Zero-cost live internet search fallback when government APIs encounter rate limits or network blocks.
* **What It Provides:** Instant HTML search snippet extraction for commodities, weather, and current events.

---

### 8. 💾 AWS DynamoDB API (`boto3`)
* **Provider:** Amazon Web Services (AWS)
* **Tables Used:**
  * `janai_calls`: Call history logs, duration, caller location, conversation turns.
  * `janai_vectors`: Vector embedding database for government schemes RAG.
  * `janai_knowledge`: Raw scheme text, eligibility, and application steps.
  * `janai_phone_profiles`: Stored user preferences (language, name, preferred agent).

---

### 9. 🗄️ AWS S3 Storage API (`boto3`)
* **Provider:** Amazon Web Services (AWS)
* **Bucket:** `janai-documents-2026`
* **Why We Use It:** Stores generated TTS WAV audio files and generates presigned HTTPS URLs for Twilio `<Play>` call playback.
